# Prompt Capture 使用指南

Prompt Capture 是一个本地优先的 AI CLI 提示词采集工具。它通过 Codex 和 Claude Code 的原生 Hook 机制捕获用户提交的 prompt，将数据保存在本机，并提供 Markdown 归档和 Web 管理界面。

![Prompt Capture Web UI](https://raw.githubusercontent.com/hekai753/prompt-capture/main/assets/web-ui.png)

## 适用场景

- 想长期归档自己在 Codex、Claude Code 中输入过的 prompt。
- 想按项目、日期、工具、事件类型检索历史输入。
- 想把 prompt 记录自动整理成 Markdown，便于回顾、复盘或沉淀工作流。
- 不想使用 terminal wrapper，也不想把数据上传到云端。

## 工作方式

Prompt Capture 不包装终端，也不读取完整 TUI 屏幕内容。它只通过 CLI 工具提供的原生 Hook 接收结构化事件：

```text
Codex / Claude Code Hook
  -> prompt-capture ingest
  -> 标准化事件
  -> 写入本地 JSONL 和查询索引
  -> 按需刷新 Markdown
  -> Web UI 浏览和管理
```

默认只采集用户提交的 prompt，也就是 `UserPromptSubmit`。工具调用和停止事件需要显式启用。

## 安装

```bash
npm install -g prompt-capture
```

确认命令可用：

```bash
prompt-capture --help
```

## 快速开始

先用 dry-run 预览将要写入的 Hook 配置：

```bash
prompt-capture install --target all --scope global --dry-run
```

安装 Claude Code Hook：

```bash
prompt-capture install --target claude --scope global
```

安装 Codex Hook：

```bash
prompt-capture install --target codex --scope global
```

启动 Web 管理界面：

```bash
prompt-capture web --port 4873
```

浏览器打开：

```text
http://127.0.0.1:4873
```

## Hook 安装说明

### 默认采集内容

默认安装只采集 `UserPromptSubmit`：

```bash
prompt-capture install --target claude --scope global
prompt-capture install --target codex --scope global
```

这会记录用户每次提交给 AI CLI 的 prompt，不采集助手回复正文。

### 采集工具事件

如需记录工具调用元数据：

```bash
prompt-capture install --target codex --scope global --events prompt,tools
```

这会启用：

```text
UserPromptSubmit
PreToolUse
PostToolUse
```

### 采集 Stop 事件

如果还想记录回合结束事件：

```bash
prompt-capture install --target codex --scope global --events prompt,tools,stop
```

### 支持的事件组合

```text
prompt              仅采集 UserPromptSubmit
prompt,tools        采集 prompt + PreToolUse + PostToolUse
prompt,tools,stop   采集 prompt + 工具事件 + Stop
all                 采集当前支持的全部 Hook 事件
```

## Codex 额外配置

Codex 需要开启用户级 Hook 功能。检查或编辑：

```text
~/.codex/config.toml
```

确保包含：

```toml
[features]
hooks = true
```

如果使用项目级 Hook，Codex 还要求项目本身已被信任。Prompt Capture 不会伪造 Codex 的 Hook trust state。

## Web 管理界面

启动：

```bash
prompt-capture web --port 4873
```

Web UI 只绑定本机地址：

```text
127.0.0.1
```

当前界面支持：

- 顶部统计：事件数、Prompt 数、项目数、当前筛选数、活跃日期、今日 Prompt、工具事件、会话数。
- 提交矩阵：展示最近 13 周 prompt 活跃情况，点击有提交的日期可筛选事件。
- 筛选：按项目、日期、来源工具、事件类型、关键词过滤。
- 日期控件：没有 prompt 的日期会置灰，不可选择。
- 事件列表：展示 prompt 摘要、工具事件和时间信息。
- 详情面板：查看完整 prompt、元数据、工具结果摘要和同会话关联事件。
- 配置弹窗：修改 raw payload 保存和 Markdown 生成模式。
- Markdown 导出：从界面触发完整 Markdown 重建。

## 配置管理

查看当前配置：

```bash
prompt-capture config get
```

默认配置：

```json
{
  "rawPayloads": false,
  "markdownMode": "realtime"
}
```

### rawPayloads

raw payload 默认关闭，减少敏感信息保留。

仅在排查适配器 payload 时启用：

```bash
prompt-capture config set rawPayloads true
```

关闭：

```bash
prompt-capture config set rawPayloads false
```

### markdownMode

默认是实时模式：

```bash
prompt-capture config set markdownMode realtime
```

实时模式下，每次采集后只刷新当前项目/日期对应的 Markdown 文件。

如果你希望只在手动导出时生成 Markdown：

```bash
prompt-capture config set markdownMode manual
```

## 数据存储位置

默认存储目录：

```text
~/.prompt-capture/
```

目录结构：

```text
~/.prompt-capture/
  config.json
  events/
    YYYY-MM-DD.jsonl
  index.json
  md/
    projects/<project-slug>/YYYY-MM-DD.md
  raw/
    <source>/YYYY-MM-DD/<event-id>.json
```

说明：

- `events/*.jsonl` 是追加式事件日志，也是最重要的源数据。
- `index.json` 是 Web UI 查询索引。
- `md/` 是可重新生成的 Markdown 归档。
- `raw/` 只有在开启 `rawPayloads` 后才会写入。

## 自定义存储目录

通过环境变量：

```bash
PROMPT_CAPTURE_HOME=/path/to/archive prompt-capture web
```

通过命令参数：

```bash
prompt-capture web --home /path/to/archive --port 4873
prompt-capture export-md --home /path/to/archive
```

## Markdown 归档

手动生成或重新生成 Markdown：

```bash
prompt-capture export-md
```

输出位置：

```text
~/.prompt-capture/md/projects/<project-slug>/YYYY-MM-DD.md
```

Markdown 是派生产物。如果删除 `md/` 目录，可以再次运行 `export-md` 从事件日志重建。

## 卸载 Hook

卸载所有 Prompt Capture 安装过的 Hook：

```bash
prompt-capture uninstall --target all --scope global
```

只卸载 Claude Code：

```bash
prompt-capture uninstall --target claude --scope global
```

只卸载 Codex：

```bash
prompt-capture uninstall --target codex --scope global
```

卸载只会删除带有 `--installed-by prompt-capture` 标记的 Hook，不会删除其他用户 Hook。

## 隐私建议

- 数据默认只保存在本机。
- 不要把 `~/.prompt-capture/` 直接提交到 Git。
- raw payload 默认关闭，除非调试不要开启。
- Prompt 可能包含密钥、内部代码、业务上下文，分享 Markdown 前应先检查内容。
- MVP 不采集助手回复正文。

## 常见问题

### Web 页面没有数据

先检查是否已有事件文件：

```bash
find ~/.prompt-capture -maxdepth 5 -type f | sort
```

如果没有 `events/YYYY-MM-DD.jsonl`，说明 Hook 尚未成功触发。

### Claude Code 或 Codex 报 Hook 输出格式错误

Prompt Capture 的 `ingest` 默认不输出 stdout。如果你手动改过 Hook 命令，确认没有额外输出。

正确形式类似：

```bash
prompt-capture ingest --source claude-code --installed-by prompt-capture
prompt-capture ingest --source codex --installed-by prompt-capture
```

### Codex 没有触发 Hook

检查：

```toml
[features]
hooks = true
```

还要确认项目已被 Codex 信任。

### 想调试真实 Hook payload

临时开启 raw payload：

```bash
prompt-capture config set rawPayloads true
```

触发一次 CLI 对话后查看：

```bash
find ~/.prompt-capture/raw -type f | sort
```

调试完建议关闭：

```bash
prompt-capture config set rawPayloads false
```

### npm 发布后 README 图片不显示

确认发布包里包含 `assets/web-ui.png`：

```bash
npm pack --dry-run
```

Wiki 页面使用的是 GitHub raw 图片地址：

```text
https://raw.githubusercontent.com/hekai753/prompt-capture/main/assets/web-ui.png
```

如果仓库默认分支不是 `main`，需要替换为实际分支名。

## 升级

升级全局包：

```bash
npm install -g prompt-capture@latest
```

升级后建议查看帮助：

```bash
prompt-capture --help
```

如果 Hook 命令格式有变化，可以重新安装 Hook：

```bash
prompt-capture install --target all --scope global
```

安装器会先移除旧的 Prompt Capture 自有 Hook，再写入当前版本的 Hook。
